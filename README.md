# 手ざわり手帳

**スマホなのに、紙。** スマホに生の質感の手帳を埋め込んだ、見た目・手触りで勝つ手帳アプリ。study-plan-app（一週間の計画帳）とは**別アプリ**。

- 名前の由来: コンセプト「生の質感」を誰でも分かる言葉にしたもの（2026-07-13ユーザー決定）
- 副題（ストア用キャッチ）: 「スマホなのに、紙。」
- 旧仮称: かみの計画帳 → 生手帳（いずれも没）

## 収益化方針（2026-07-13ユーザー決定）

- **広告を入れる**。形式は「挟み込みチラシ」: ページをめくるとたまにページ間に全画面広告が挟まっている（めくって飛ばせる。頻度制限: 例として1日初回＋20めくりに1回まで）。補助として最下部に新聞下段風の小さな紙面広告枠
- **課金（表紙購入 or サブスク）でチラシが挟まらなくなる**。ただし「広告なし」を宣伝文句にはしない（study-plan-appと同方針）
- 文具課金（表紙・ペン・マステ・スタンプ）と広告の二本柱
- 広告SDK（AdMob想定）の実装はネイティブ化時。Web版には入れない

## コンセプトの核（2026-07-13ユーザーの言語化）

**「デジタルカレンダーの上に被せる、紙の顔」**。予定データはGoogle/iCloudに置いたまま、毎日のページに「印刷」されて届く。その上に自分の字で書くのは計画・日記・しるし。紙手帳派の「二重記帳問題」（スマホと紙に同じ予定を2回書く）を構造的に解決する。

- 設計の線引き: **印刷されたもの（予定）は触れない・書いたもの（計画/日記/インク)は自分のもの**。予定の編集機能は足さない（それはカレンダーアプリの仕事）
- コピー候補: 「予定はカレンダーに。気持ちは手帳に。」「あなたのカレンダーが、毎朝1ページの紙になる。」

## プロダクト仮説

- 手帳・ジャーナリング市場は「文房具を買う感覚」の課金（表紙・ペン・マステ・スタンプ）が効く
- 機能で勝つ計画帳（study-plan-app）に対し、こちらは**愛着と手触りで勝つ**
- 収益化はアイテム課金＋サブスク想定（広告よりも世界観に合う）

## v1プロトタイプに入っているもの

- 紙の質感（SVGノイズの粒・罫線・赤マージン線・綴じステッチ・ページの束の影）
- 1日1ページ、スワイプ or ボタンで**3Dページめくり**（rotateY、紙の裏面も描画）
- ✏️ペンモード: **指で書ける鉛筆**（速度で太さ・濃さが変わる、かすれ表現、消しゴム付き）
- キーボード書き（手書きフォントKlee Oneで罫線に沿う）
- 日付ヘッダー（和暦付き）
- **日記欄**（2026-07-13追加）: 日ページの下側が「日記」区画。左余白に区画名が印刷され、少し濃い罫線で区切られる。計画=pt:ISO:text / 日記=pt:ISO:diary で別保存。どちらか書けば月カレンダーにしるしが付く
- localStorage保存（`pt:<ISO>:text` / `pt:<ISO>:ink`=ベクター形式 {v,cw,ch,strokes,base?}。線は座標×10＋間隔msの列で、開くたびに同じ描き味で再生。1ページ数百バイト。旧PNG形式はbaseとして下に敷いて引き継ぐ）

## 技術

- 単一 `index.html`・vanilla JS・依存なし（フォントのみGoogle Fonts。製品化時はバンドルする）
- iPhone Safari想定（study-plan-appと同じ流儀）

## 手書きの位置づけ（2026-07-13ユーザー決定）

手書きは「文章を書く手段」ではなく**「しるし・飾り」**（丸をつける、線を引く、落書き）。文章はキーボード＋手書きフォント。この前提なら1ページの線は数本で、ベクター保存と噛み合う。赤鉛筆・マーカー等の課金文具もこの用途の延長。

## 実装済みの追加機能（2026-07-13）

- **縁なし表示**: スマホでは紙が画面いっぱい（埋め込みモード）。PCは机ビュー
- **オフライン対応**: Service Worker（シェル事前キャッシュ＋フォントは使った字から貯まる）
- **月カレンダー**: 右端のインデックスタブ「日/月」でめくり替え。書いた日に鉛筆のしるし、今日は赤鉛筆の丸、日付タップでその日へ。月ページのスワイプで前後の月
- **Googleカレンダー連携は全カレンダー統合**（2026-07-13追記）: calendarList（24hキャッシュ、pt:gcal-cals）から表示中の全カレンダーを取得して日ごとに統合・時刻ソート（終日が先頭）。Googleの祝日カレンダーは印刷済みの祝日と二重になるため除外。旧スコープ連携は403→primaryのみにフォールバックし設定に再連携の案内（pt:gcal-scope-old）
- **Googleカレンダー連携（読み取り）**: 一週間の計画帳と同じWorker基盤（/google/code・/google/token、refresh tokenはKV側）を流用。同一オリジンでlocalStorageを共有するためキーは pt: 接頭辞で分離。予定は日ページ上部に「印刷」風に表示され、書き込み欄が罫線単位で下にずれる。日ごとに10分キャッシュ（オフラインでも直近の予定が見える）。**iCloudカレンダーはネイティブ化時にEventKitで追加する方針（2026-07-13決定。EventKitなら端末の全カレンダーが1本で取れて重複も構造的に起きない）**
- ビュー構造: currentView = day:YYYY-MM-DD | month:YYYY-MM（stepViewが前後を解決、めくり機構は共通）

## 暦情報（2026-07-13実装）

- **祝日**: 2026〜2027年を埋め込み（振替休日・国民の休日込み）。日付が赤くなり祝日名が印刷される
- **六曜**: 旧暦の朔日テーブル（KYUREKI_LUNATIONS、公開の旧暦・六曜カレンダー複数と照合検証済み）から (旧暦月+日)%6 で計算。ページヘッダーと月カレンダーに印刷。設定でオフ可（pt:rokuyo）
- データ範囲: 2026-01-01〜2028-01下旬。**範囲外は表示しない**（誤った暦を出さないポリシー）。毎年、翌年分の朔日と祝日を追加する運用
- 二十四節気は要相談（2026-07-13時点。入れるなら同じ「印刷の細部」路線で）
